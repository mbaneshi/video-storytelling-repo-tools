# Prompt: Storytelling Frameworks for Technical Demos

## Context

I'm a developer creating short demo videos (2–5 min) for hackathon submissions, project showcases, and developer audiences. I need structured storytelling approaches — not generic marketing advice, but frameworks that work specifically for showing software to technical judges and peers.

## Research Questions

1. **Established frameworks**: What storytelling frameworks exist for technical product demos? Compare:
   - Problem → Solution → Impact (classic)
   - Hero's Journey (adapted for dev tools)
   - STAR (Situation, Task, Action, Result)
   - PAS (Problem, Agitation, Solution)
   - Before/After/Bridge
   - "Show, don't tell" demo-first approach
   - Which framework works best for which video length and audience?

2. **Hackathon-specific patterns**: Analyze winning hackathon demo videos. What patterns do top submissions follow? What do judges actually look for in the narrative structure?

3. **Pacing and information density**: For a technical audience, what's the right ratio of:
   - Live demo vs. slides/diagrams
   - Narration vs. on-screen text
   - Technical depth vs. high-level overview
   - How does this change between 2-min and 5-min formats?

4. **Emotional hooks for technical content**: How do you create engagement in a dev tool demo without being cheesy? What emotional triggers work for developer audiences (frustration with status quo, elegance of solution, "wow" moment)?

5. **Script structures**: Give me 3 concrete script templates I can fill in for:
   - A CLI tool demo (2 min)
   - An AI agent project demo (3 min)
   - A full-stack app demo (5 min)

## What I Want Back

- Framework comparison with pros/cons for each
- Real examples from winning hackathon demos if possible
- Ready-to-use script templates
- Anti-patterns to avoid (what makes a demo boring or confusing)
